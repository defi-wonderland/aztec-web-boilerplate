# Aztec Web Boilerplate

A starter template for building web applications on Aztec Network. This boilerplate provides a complete setup for developing privacy-preserving dApps with React, Vite, and Aztec.js.

## Overview

This boilerplate demonstrates:
- Privacy-preserving token operations using Aztec's private state
- Embedded wallet with in-browser key generation
- Azguard wallet integration for external wallet support
- Sponsored fee payments through SponsoredFPC
- Network switching between local sandbox and testnet

## Quick Start

### Prerequisites
- Node.js >=22.0.0
- [Aztec CLI](https://docs.aztec.network/getting-started) installed

### Installation

```bash
# Clone the repository
git clone https://github.com/your-org/aztec-web-boilerplate.git
cd aztec-web-boilerplate

# Install dependencies
yarn install

# Install Aztec toolchain (if not already installed)
aztec-up

# Start the Aztec sandbox
aztec start --sandbox

# Build and deploy contracts (in another terminal)
yarn build-contracts
yarn deploy-contracts

# Start the development server
yarn dev
```

The application will be available at http://localhost:3000

## Configuration

### Deployment Config Files

Contract deployment addresses are stored in JSON config files under `src/config/deployments/`:

```
src/config/deployments/
├── sandbox.json   # Local sandbox deployment (generated by deploy script)
├── testnet.json   # Testnet deployment (generated by deploy script)
├── types.ts       # TypeScript types for deployment config
└── index.ts       # Config loaders
```

When you run `yarn deploy-contracts`, the script automatically updates the appropriate config file with:
- Contract addresses
- Deployment salts
- Deployer address
- Network-specific settings

These config files can be committed to version control to share deployment addresses with your team.

### Environment Variables (Optional)

You can override default settings using environment variables. Copy `.env.example` to `.env`:

```bash
cp .env.example .env
```

Available overrides:
- `VITE_AZTEC_NODE_URL` - Override the Aztec node URL
- `VITE_PROVER_ENABLED` - Enable/disable prover (false = faster development)
- `AZTEC_NODE_URL` - Node URL for deployment scripts
- `PROVER_ENABLED` - Prover setting for deployment scripts

### Network Switching

The app supports multiple networks:
- **Sandbox** - Local development (default)
- **Testnet** - Public test network
- **Custom** - User-defined network via UI

Deploy to different networks:
```bash
# Deploy to local sandbox (default)
yarn deploy-contracts

# Deploy to testnet
yarn deploy-contracts:testnet
```

## Development

### Essential Commands

```bash
# Contract Development
yarn build-contracts      # Compile Noir contracts and generate TypeScript artifacts
yarn compile-contracts    # Compile Noir contracts only
yarn codegen-contracts    # Generate TypeScript bindings
yarn deploy-contracts     # Deploy to sandbox
yarn deploy-contracts:testnet  # Deploy to testnet

# Application Development  
yarn dev                  # Start development server
yarn build-app           # Production build
yarn build               # Full build (contracts + app)
yarn serve               # Serve production build

# Testing & Quality
yarn test                # Run test suite
yarn test:unit          # Run unit tests only
yarn test:integration   # Run integration tests only
yarn test:e2e           # Run E2E tests with Playwright
yarn lint                # Check code formatting
```

### Fast Development Mode

For faster iteration, disable the prover:

```bash
# One-time
PROVER_ENABLED=false yarn deploy-contracts

# Or set in .env
echo "PROVER_ENABLED=false" >> .env
```

## Architecture

### Aztec Integration
- **PXE**: Client-side Private eXecution Environment for proof generation
- **Account Contracts**: Schnorr signature-based accounts
- **Note System**: Encrypted UTXO-like notes for private state
- **Fee Abstraction**: Sponsored transactions through SponsoredFPC

### Wallet Options
1. **Embedded Wallet**: Keys generated and stored in browser
2. **Azguard Wallet**: External wallet integration via browser extension

## Customization

### Adding New Contracts

1. Create contract in `contracts/your_contract/`
2. Add to `contracts/Nargo.toml` workspace
3. Run `yarn build-contracts`
4. Update `scripts/deploy.ts` to deploy your contract
5. Import generated TypeScript bindings from `src/artifacts/`

### Adding New Networks

1. Create `src/config/deployments/your_network.json` with placeholder values
2. Create loader in `src/config/networks/your_network.ts`
3. Add to `AVAILABLE_NETWORKS` in `src/config/networks/index.ts`
4. Update deploy script to support the new network
